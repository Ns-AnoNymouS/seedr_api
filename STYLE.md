# Coding Style Guide ‚Äî `seedr-api`

A reference for every contributor. These rules are **enforced automatically** where possible (ruff, mypy) or are **conventions** (naming, docstrings, git).

---

## 1. Python Version & Language

- **Minimum**: Python 3.10. Use `|` union types, `match`, `TypeAlias`, `ParamSpec`, etc. freely.
- Always add `from __future__ import annotations` as the **first non-comment line** in every module. This enables PEP 563 postponed evaluation of annotations and keeps 3.10-style unions working everywhere.
- Prefer built-in generic types: `list[int]`, `dict[str, Any]`, `tuple[str, ...]` ‚Äî **not** `List`, `Dict`, `Tuple` from `typing`.
- Use `collections.abc` for abstract types in signatures: `Sequence`, `Mapping`, `AsyncGenerator`, `Callable`, etc.

## 2. Formatting & Linting

All formatting and lint rules are enforced by **ruff**.

```
ruff check src/ tests/   # lint
ruff format src/ tests/  # format (black-compatible)
```

| Setting | Value |
|---------|-------|
| Max line length | 88 chars |
| Target version | `py310` |
| Import sorter | isort via ruff (`I`) |

Active ruff rule sets: `E W F I UP B C4 ANN S RUF`

> [!TIP]
> Run `ruff check --fix src/ tests/` to auto-fix safe violations before committing.

### Per-file overrides
- `tests/*` ‚Äî `ANN` and `S` are disabled (annotations not required in tests, `assert` is fine).
- `src/seedr_api/__init__.py` ‚Äî `ANN` is disabled (re-export modules need no annotation).

## 3. Type Annotations

- **All public functions must have full type annotations** (enforced by `ANN` ruff rules and `mypy --strict`).
- Private helpers (leading `_`) must also be annotated.
- Never use `Any` unless the value is genuinely opaque (e.g. raw JSON from an uncontrolled API); annotate the variable explicitly:
  ```python
  data: Any = await self._http.get("/endpoint")   # OK ‚Äî raw JSON
  ```
- Avoid `Optional[X]`; use `X | None` instead.
- Avoid `Union[X, Y]`; use `X | Y` instead.
- Return type must always be specified; use `-> None` for procedures.
- Prefer `TypeAlias` for complex repeated types:
  ```python
  JsonObject: TypeAlias = dict[str, Any]
  ```

### mypy

```
mypy src/seedr_api/ --strict
```

Config lives in `[tool.mypy]` in `pyproject.toml`. The `pydantic.mypy` plugin is active ‚Äî do not disable it.

## 4. Docstrings

Use **NumPy-style** docstrings for all public classes, methods, and functions.

```python
def example(arg: int, *, flag: bool = False) -> str:
    """One-line summary sentence (imperative mood, no trailing period).

    Extended description of behavior, edge cases, or important notes.
    May span multiple paragraphs.

    Parameters
    ----------
    arg:
        Description of the first parameter.
    flag:
        Description of the keyword-only flag.

    Returns
    -------
    str
        What the return value represents.

    Raises
    ------
    ValueError
        When the argument is out of range.

    Example
    -------
    ::

        result = example(42, flag=True)
    """
```

Rules:
- Every **public** class and method must have a docstring.
- Private helpers (`_name`) should have a one-line docstring at minimum.
- Do **not** document the `self` parameter.
- Use backtick cross-references: `:class:\`ClassName\``, `:meth:\`method\``.

## 5. Naming Conventions

| Construct | Convention | Example |
|-----------|-----------|---------|
| Module | `snake_case` | `_http.py`, `filesystem.py` |
| Class | `PascalCase` | `SeedrClient`, `AsyncHTTPClient` |
| Public function / method | `snake_case` | `list_root_contents()` |
| Private method | `_snake_case` | `_build_headers()` |
| Module-level constant | `ALL_CAPS` | `_DEFAULT_TIMEOUT` |
| Type alias | `PascalCase` | `JsonObject` |

- Prefix internal constants with `_` to signal they are not public API.
- Resource class names end in `Resource`: `FilesystemResource`, `TasksResource`.
- Model class names are plain nouns: `FileInfo`, `Task`, `TokenResponse`.

## 7. Async / Concurrency Patterns

- Every method that makes a network request **must** be `async`.
- Sync wrappers around async calls are **not** provided (use `asyncio.run()` externally).
- **Never** use blocking I/O inside a coroutine. Offload to `asyncio.to_thread`:
  ```python
  # ‚úÖ correct
  file_bytes = await asyncio.to_thread(path.read_bytes)

  # ‚ùå wrong ‚Äî blocks the event loop
  file_bytes = path.read_bytes()
  ```
- Async context managers must implement `__aenter__` / `__aexit__`.
- Use `@asynccontextmanager` from `contextlib` for generator-based async context managers.
- Streaming responses must use `AsyncGenerator[bytes, None]` ‚Äî do not buffer entire responses in memory.

## 8. Pydantic Models

- All API response models inherit from `pydantic.BaseModel`.
- Use `model_validate(data)` ‚Äî not the deprecated `parse_obj`.
- Field defaults that are mutable **must** use `Field(default_factory=...)`:
  ```python
  # ‚úÖ correct
  tags: list[str] = Field(default_factory=list)

  # ‚ùå wrong ‚Äî shared mutable default
  tags: list[str] = []
  ```
- Optional API fields use `X | None = None`, not bare defaults.
- Use `Field(description="...")` when the semantics of a field are not obvious from the name.
- Datetime fields from the API must stay as `str` until the API guarantees a stable format; parse them in application code if needed.
- Do **not** add validators that silently coerce unexpected data ‚Äî prefer raising `ValidationError`.

## 9. Error Handling

Exception hierarchy:

```
SeedrError (base)
‚îú‚îÄ‚îÄ AuthenticationError   HTTP 401
‚îú‚îÄ‚îÄ ForbiddenError        HTTP 403
‚îú‚îÄ‚îÄ NotFoundError         HTTP 404
‚îú‚îÄ‚îÄ RateLimitError        HTTP 429  (+ retry_after: int | None)
‚îú‚îÄ‚îÄ ServerError           HTTP 5xx
‚îî‚îÄ‚îÄ APIError              Other 4xx
```

Rules:
- **Never** use bare `except Exception` or `except BaseException`. Catch the narrowest possible exception.
- When parsing an uncertain response body, catch only `(ValueError, aiohttp.ContentTypeError)`.
- Map every expected HTTP status code to a typed exception in `_http.py:_raise_for_status`.
- Preserve the original `status_code` in every `SeedrError` subclass.
- Re-raise third-party exceptions as `SeedrError` subclasses if they cross the public API boundary.

## 10. Testing

- Framework: `pytest` + `pytest-asyncio` (`asyncio_mode = "auto"` ‚Äî **no** `@pytest.mark.asyncio` needed).
- HTTP mocking: `aioresponses`.
- Coverage: `pytest-cov`, minimum **80%** (`fail_under = 80`).
- Type-annotate all fixtures and test functions.
- Tests live under `tests/` mirroring the `src/seedr_api/` structure.
- Fixture return types for `yield`-based fixtures must be `Generator[T, None, None]` or `AsyncGenerator[T, None]`.

### Test naming

```python
def test_<unit>_<scenario>_<expected_outcome>() -> None:
    ...
```

Example: `test_filesystem_get_root_returns_folder_info`

### Test structure

Follow **Arrange ‚Üí Act ‚Üí Assert**:

```python
async def test_tasks_add_magnet_returns_task(
    token_client: SeedrClient,
    mock_aioresponses: aioresponses,
) -> None:
    # Arrange
    mock_aioresponses.post(re.compile(r".*/tasks"), payload={"id": 1, "name": "File"})

    # Act
    task = await token_client.tasks.add_magnet("magnet:?xt=...")

    # Assert
    assert task.id == 1
    assert task.name == "File"
```

## 11. Imports

Order (enforced by ruff/isort):

1. `__future__`
2. Standard library
3. Third-party (`aiohttp`, `pydantic`)
4. First-party (`seedr.*`)

One blank line between each group. No wildcard imports (`from x import *`).

## 12. Git & Commits

- **Branch naming**: `feat/<short-desc>`, `fix/<short-desc>`, `docs/<short-desc>`, `chore/<short-desc>`
- **Commit messages** follow [Conventional Commits](https://www.conventionalcommits.org/):
  ```
  feat(tasks): add add_url() method for direct torrent links
  fix(http): narrow bare except to ContentTypeError in _raise_for_status
  docs(readme): fix incorrect method name in Quick Start example
  ```
- Keep commits atomic ‚Äî one logical change per commit.
- Do **not** commit generated files (`__pycache__`, `.mypy_cache`, `*.egg-info`, `dist/`).

## 13. Release & Versioning

Follows the [PyPA Packaging Tutorial](https://packaging.python.org/en/latest/tutorials/packaging-projects/).

**`pyproject.toml` requirements (PyPA standard):**

```toml
[build-system]
requires = ["hatchling>=1.26", "hatch-vcs"]   # >=1.26 required for PEP 639 license metadata
build-backend = "hatchling.build"

[project]
license = "MIT"                                 # SPDX expression (PEP 639) ‚Äî NOT the old table form
license-files = ["LICEN[CS]E*"]                 # glob ‚Äî lets build tools bundle LICENSE automatically
classifiers = [
    "Operating System :: OS Independent",       # always required per PyPA tutorial
    ...
]
```

> [!IMPORTANT]
> Do **not** use the old `license = { file = "LICENSE" }` table form ‚Äî it is deprecated per PEP 639 and not supported by `hatchling >= 1.26`.

- Version is managed by `hatch-vcs` (git tags). **Do not** edit `_version.py` manually.
- Releases are tagged `vX.Y.Z` (semver).
- Update `CHANGELOG.md` before tagging a release.
- Build and verify locally before publishing:

```bash
pip install hatch build
python -m build          # produces dist/*.whl and dist/*.tar.gz
```

- Publish with `twine` or `hatch publish`:

```bash
pip install twine
twine upload dist/*
```
